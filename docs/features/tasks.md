## The Problem

Task creation is manual, reactive, and disconnected from reality. You create tasks based on what you *think* needs doing — not what the data actually shows. Important work gets missed because nobody translated that stale PR or failing pipeline into a task.

---

## The Solution

<div class="screenshot" markdown>
![Tasks — Kanban board with AI-driven task creation](../assets/screens/flavors/pm/tasks.png)
</div>

Ask JP to *"Create tasks based on current project priorities"* and watch it analyze your live project data — commits, PRs, sprint risks, blockers — then generate a prioritized task list that reflects what actually needs attention.

!!! success "Impact"
    **Your task list finally reflects reality.** AI sees what humans miss — the stale PR that's blocking two teams, the sprint risk that needs mitigation today.
